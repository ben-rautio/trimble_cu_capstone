void LEUART0_IRQHandler ( void  )

Interrupt Handler for LEUART0.

A variable int_flag will store any interrupt flag if it has also been enabled. All of the interrupts will then be cleared from LEUART0->IF. The function handles the scheduling of processing each interrupt.

Note
Int_flag will be used to service every interrupt, with the priority given to txbl, then txc.

LEUART App Receive Byte polls a receive byte for the LEUART TDD test.

The BLE module will respond to AT commands if the BLE module is not connected to the phone app. To validate the minimal functionality of the LEUART peripheral, write and reads to the LEUART will be performed by polling and not interrupts.

Note
In polling a receive byte, a while statement checking for the RXDATAV bit in the Interrupt Flag register is required before reading the RXDATA register.

Initializes the LEUART peripheral for functionality.

Takes in pointers to leuart and leuart_settings struct to initialize the leuart peripheral. Enables the clock to the leuart peripheral and passes the values from leuart_settings to the leuart peripheral.

Note
Due to the amount of asynchronous registers of the LEUART, while syncbusy loops are utilized to ensure a portion of the leuart is configured before configuring the next portion. Does not enable any interrupts, and only enables the LEUART NVIC at the end of the function.
Parameters
[in]*leuartA pointer to the base addres of the LEUART peripheral being used.
[in]*leuart_settingsA pointer to the base address of the settings struct for the LEUART functionality.

Initializes the LEUART0 State Machine for bluetooth transmission.

After initializing the State Machine, enable the txbl interrupt and exit the critical state. This will drive the board into the LEUART0 interrupt handler and start sending each byte of the string.

Note
This function is made atomic by CORE CRITICAL calls.
Parameters
[in]*leuartA pointer to the base address of one of the leuart peripherals.
[in]*stringA pointer to the string being sent to the bluetooth device.
[in]string_lenThe length of the string being sent.

The TXBL handler for the LEUART state machine.

The switch statement in the function will either send bytes through the leuart's transmit register or switch the state of the state machine depending on when the TXBL interrupt occurs.

Note
If a TXBL interrupt occurs in the DONE or default states, then an EFM_ASSERT will be asserted as false.

◆ leuart_txcIRPT()

void leuart_txcIRPT ( void  )
static

LEUART driver.

The TXC handler for the LEUART state machine.

This module contains all the functions to support the driver's state machine to transmit a string of data across the LEUART bus. There are additional functions to support the Test Driven Development test that is used to validate the basic set up of the LEUART peripheral. The TDD test for this class assumes that the LEUART is connected to the HM-18 BLE module. These TDD support functions could be used for any TDD test to validate the correct setup of the LEUART.

The switch statement in the function will only have functionality for the DONE state. In this state, the TXC interrupt will unblock EM3 sleep mode, add the scheduled event to the scheduler, and reset the state machine.

Note
If a TXC interrupt occurs in any state besides the DONE state, then an EFM_ASSERT statement will be asserted to false; indicating a false state.
